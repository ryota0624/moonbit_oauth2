## Issue報告ガイドライン

新しい技術的問題を発見した場合、以下のテンプレートを使用してドキュメント化してください：

### ファイル名
`YYYYMMDD_issue_short_description.md`

### テンプレート

```markdown
# Issue: [問題の簡潔な説明]

## 概要
[問題の1-2行の要約]

## 環境
- MoonBit バージョン:
- OS:
- 関連パッケージとバージョン:

## 問題の詳細

### 症状
[何が起きているか]

### 再現手順
1. [ステップ1]
2. [ステップ2]
...

### 期待される動作
[本来どうあるべきか]

### 実際の動作
[実際に何が起きているか]

## 関連コード
[問題に関連するコードスニペット]

## 試行した解決策
1. [試したこと1] → [結果]
2. [試したこと2] → [結果]

## 分析
[原因の推測、証拠]

## 回避策
[現在利用可能な回避策]

## 次のアクション
- [ ] [アクション1]
- [ ] [アクション2]

## 参考情報
- [関連リンク]
- [外部リソース]

## ステータス
- 発見日: YYYY-MM-DD
- 最終更新: YYYY-MM-DD
- 状態: [未解決 / 調査中 / 解決済み / 回避策あり]
- 優先度: [高 / 中 / 低]
```

## ステータス定義

- **未解決**: 問題が存在し、解決策が見つかっていない
- **回避策あり**: 問題は残っているが、作業を続けられる回避策がある
- **調査中**: 原因を調査している段階
- **解決済み**: 問題が完全に解決された
- **クローズ**: 問題が無効化されたか、対応不要と判断された

## 優先度定義

- **高**: 作業をブロックする重大な問題
- **中**: 回避策はあるが、対応が望ましい問題
- **低**: マイナーな問題、または将来的な改善事項
